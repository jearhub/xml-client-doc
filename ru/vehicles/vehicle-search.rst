Поиск Авто
==========

Описание XML схем поиска авто
=============================

XSD-схемы поиска авто:

:download:`www/xsd/vehicle/VehicleSearchRequest.xsd <../../themes/hotelbook/static/xsd/vehicle/VehicleSearchRequest.xsd>`

:download:`www/xsd/vehicle/VehicleSearchResponse.xsd <../../themes/hotelbook/static/xsd/vehicle/VehicleSearchResponse.xsd>`

Асинхронный поиск авто - см. `Элемент SearchId <#h1285-20>`_

Запрос на поиск авто, VehicleSearchRequest
------------------------------------------

XML-схема запроса:

::

    <?xml version="1.0" encoding="utf-8"?>

    <VehicleSearchRequest>
      <Request    
       [vehicleClass="all"]/>
       [vehicleType="all"]/>
       [vehicleTransmission="all"]/>   
       [vehicleAC="all"]/>   
       [seats="0"]/>   
       [confirmation="all"]/>   
      <Providers>
         <ProviderId>..</ProviderId>
      </Providers>
      <PickUp
        cityId="id"    
       [airportId="id"]
       date="YY-mm-dd"
       hour="HH:ii"/>
      <DropOff
        cityId="id"    
       [airportId="id"]
       date="YY-mm-dd"
       hour="HH:ii"/>
    </VehicleSearchRequest>

Элемент Request
---------------

В элементе указываются параметры запроса (как аттрибуты).

- Обязательный элемент.
- Дочерних элементов нет.

Аттрибуты элемента ``Request``:

+---------------------------+--------------------------+--------------------+------------------------------------------------------------------------------------------------------------------------+
| **Аттрибут**              | **Тип**                  | **Обязательный**   | **Описание**                                                                                                           |
+---------------------------+--------------------------+--------------------+------------------------------------------------------------------------------------------------------------------------+
| ``vehicleClass``          | строка                   | нет                | id класса авто, список всех доступных значений /xml/vehicle\_classes; Если all или не указан – независимо              |
+---------------------------+--------------------------+--------------------+------------------------------------------------------------------------------------------------------------------------+
| ``vehicleType``           | строка                   | нет                | id типа авто, список всех доступных значений /xml/vehicle\_types; Если all или не указан – независимо                  |
+---------------------------+--------------------------+--------------------+------------------------------------------------------------------------------------------------------------------------+
| ``vehicleTransmission``   | строка                   | нет                | id трансмиссии авто, список всех доступных значений /xml/vehicle\_transmissions; Если all или не указан – независимо   |
+---------------------------+--------------------------+--------------------+------------------------------------------------------------------------------------------------------------------------+
| ``vehicleAC``             | число 0|1                | нет                | id транспорта, список всех доступных значений /xml/vehicle\_fuel\_ac; Если all или не указан – независимо              |
+---------------------------+--------------------------+--------------------+------------------------------------------------------------------------------------------------------------------------+
| ``seats``                 | число                    | нет                | Кол-во мест в авто                                                                                                     |
+---------------------------+--------------------------+--------------------+------------------------------------------------------------------------------------------------------------------------+
| ``confirmation``          | online, onRequest, all   | нет                | Требуемый вид подтверждения. Если all или не указан – независимо                                                       |
+---------------------------+--------------------------+--------------------+------------------------------------------------------------------------------------------------------------------------+

Элемент Providers
-----------------

Список поставщиков, по которым будет производиться поиск (указываются как вложенные элементы ``ProviderId``).
Этот элемент имеют право указывать только те пользователи, у которых установлено право "Видит поставщика".

- Необязательный элемент.
- Аттрибутов нет.

Дочерние элементы:

+------------------+--------------------+------------------------------------------------------------------------------------------------------+
| **Элемент**      | **Обязательный**   | **Описание**                                                                                         |
+------------------+--------------------+------------------------------------------------------------------------------------------------------+
| ``ProviderId``   | да                 | id поставщика, постовщиков может быть несколько, список всех доступных поставщиков: /xml/providers   |
+------------------+--------------------+------------------------------------------------------------------------------------------------------+

Ответ на поиск авто, VehicleSearchResponse
------------------------------------------

XML-схема ответа:

::

    <?xml version="1.0" encoding="utf-8"?>

    <VehicleSearchResponse>
      <VehicleSearchRequest>... исходный запрос ...</VehicleSearchRequest>
      <Errors>
        <Error code="..." description="..."> - ошибки
      </Errors>

      <VehicleSearch searchId="id" >
         <SearchPickUp  
        countryId="id"
        countryName="..." 
        cityid="id"
        cityName="..." />
        [airportid="id"] 
        [airportName="..."] /> - получение
         <SearchDropOff  
        countryId="id"
        countryName="..." 
        cityid="id"
        cityName="..." />
        [airportid="id"] 
        [airportName="..."] /> - возврат
      </VehicleSearch >
            
      <Vehicles>
        <Vehicle
        resultId="id"
        vehicleName="..." 
        [providerId="id"] -- поля может не быть, в зависимости от настроек пользователя
        confirmation="onRequest|online|inaccessible"  -- вид подтверждения    
        specialOffer="id"  -- спецпредложение    
        >
        <VehicleClass id=".." > </VehicleClass>
        <VehicleType id=".." > </VehicleType>
        <VehicleCompany id=".." logo_url=".." > </VehicleCompany>
        <ACRISS 
           acrissClass="id" 
           acrissType="id"  
           acrissTransmission="id" 
           acrissFuelAC="id" > 
        </ACRISS>
        <VehicleImage url=".." />
        <VehicleInformation 
           doors=".." 
           auto=".."  
           ac=".." 
           luggagelarge=".." />
        <SpecialOffer 
           acrissClass="id" />
        </SpecialOffer>      
        <VehicleInsurancePolicy 
          id=".." 
          name=".."  
          price=".." 
          currency=".." 
          comparePrice=".."        
          currencyTarget=".." > 
        </VehicleInsurancePolicy>      
        </Vehicle>
      </Vehicles>
    </VehicleSearchResponse>

В случае, если запрашиваются авто асинхронным способом (с указанием
параметра ?async=1), то элемент будет содержать id инициализированного
поиска. Данный ответ придет сразу, на стороне Хотелбука фоново будет
происходить поиск авто. Найденные авто можно запрашивать периодически
запросом ``/xml/vehicle_search_async?login=&search_id=&from_result_id=``
(например каждую секунду). Более подробно см. `vehicle-search-async.html <vehicle-search-async.html>`_
Необязательный элемент. Может отсутствовать, если возникли ошибки.
